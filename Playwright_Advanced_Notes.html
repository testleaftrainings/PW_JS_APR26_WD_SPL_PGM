6. What happens internally if multiple dialog listeners are attached?

## Theory

All listeners execute for same dialog event.

Can cause:

• duplicate actions

• synchronization conflicts

• unexpected failures

## Wrong Code

```ts id="rjlwm1"

class="keyword">await d.accept();

});

page.on('dialog', class="keyword">async d => {
class="keyword">await d.dismiss();

});

```

## Real-Time Scenario

Framework utility and test both added listeners.

Execution failed inconsistently.

## Explanation

Single centralized listener preferred.

## Quick Recap

“Avoid multiple listeners for same browser event.”
